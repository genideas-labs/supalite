SupaLite is a lightweight TypeScript PostgreSQL client with the Supabase query builder you already know — query builder, RPC, and transactions, with a smaller footprint and less overhead in production.

$ npm install supalite
Get started GitHub npm

Used in production by oqoq.ai.

A slim Supabase client for query builder + RPC + transactions. Need full Supabase features like auth, storage, or realtime? Use supabase-js. SupaLite stays focused on being fast and small for the database layer.

Everything you need for the query layer

{ }

Type-safe query builder

Supabase-style chainable API, written in TypeScript with full type support across tables, views, functions, and enums.

⇄

Transactions

Safe, concurrency-safe DB transactions — something the Supabase client doesn't support. Each call runs on its own pooled connection.

ƒ

RPC

Call stored procedures and functions, including single() and maybeSingle() result helpers and scalar returns.

⊞

Multi-schema

Work across multiple schemas in a single typed client, with views, functions, and enums fully typed.

⚡

Performance

Connection pooling and efficient SQL generation. No HTTP/PostgREST serialization overhead between you and Postgres.

⌁

Type generation CLI

supalite gen types generates types from your live database — a superset of Supabase CLI output, with byte-for-byte compatibility on demand.

Reads like Supabase

Keep the query ergonomics you already know. Here's the same query — fetch active users, newest first, page 2 (10 per page) — in each tool.

typescript
const { data } = await client
  .from('users')
  .select('id, name, email, created_at')
  .eq('status', 'active')
  .order('created_at', { ascending: false })
  .limit(10)
  .offset((page - 1) * 10);
typescript
const data = await prisma.user.findMany({
  select: { id: true, name: true, email: true, created_at: true },
  where: { status: 'active' },
  orderBy: { created_at: 'desc' },
  take: 10,
  skip: (page - 1) * 10,
});
typescript
const data = await db
  .select({ id: users.id, name: users.name, email: users.email, created_at: users.createdAt })
  .from(users)
  .where(eq(users.status, 'active'))
  .orderBy(desc(users.createdAt))
  .limit(10)
  .offset((page - 1) * 10);

What's supported

Supported

  • Select, filters, order, pagination
  • PostgREST-style embeds (related(*), !inner)
  • Insert, update, delete, upsert (incl. ignoreDuplicates)
  • RPC (incl. single / maybeSingle)
  • Concurrency-safe transactions
  • Multi-schema, views, functions, enums

Not supported — use supabase-js

  • Auth
  • Storage
  • Realtime

SupaLite vs Prisma / Drizzle

Choose SupaLite when you want a thin, SQL-close query layer — especially when migrating off Supabase — and handle migrations separately. Choose Prisma or Drizzle when you want schema-first modeling and built-in migrations.

Quickstart

  1. 1Install

    bash
    npm install supalite

  2. 2Connect

    typescript
    import { SupaLitePG } from 'supalite';
import { Database } from './types/database';

const client = new SupaLitePG<Database>({
  connectionString: process.env.DB_CONNECTION,
});

  3. 3Query & transact

    typescript
    const { data, error } = await client
  .from('users')
  .select('id, name, email')
  .eq('status', 'active')
  .order('created_at', { ascending: false })
  .limit(10);

// Transactions — not supported by the Supabase client
await client.transaction(async (tx) => {
  const { data: user } = await tx
    .from('users')
    .insert({ name: 'Hong', email: 'hong@example.com' })
    .select()
    .single();

  await tx
    .from('profiles')
    .insert({ user_id: user.id, bio: 'Transaction profile' });
});

  4. 4Generate types from your database

    bash
    npx supalite gen types \
  --db-url "postgresql://user:pass@localhost:5432/db" \
  --out database.types.ts
